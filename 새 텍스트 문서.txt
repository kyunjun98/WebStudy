웹 페이지
 - 정적페이지: HTML, CSS, JS(JavaScript), image
	      데이터베이스에서 정보를 가져오거나 별도의 서버에서의 처리가 없어도
	      사용자들에게 보여 줄수 있는 페이지
	      어떠한 사용자가 와도 동일한 페이지를 보여준다.
	      항상 동일한 페이지를 반환한다.

 - 동적페이지: JSP
	      서버에서 데이터베이스에서 정보를 가져와서 처리하는 것처럼, 어떠한 요청에 의하여
	      서버가 일을 수행하고 해당 결과가 포함된 파일을 보여주는 페이지
	      사용자들마다 다른 페이지가 보여질 수 있다.
	      웹 서버에 의해서 실행되는 프로그램을 통해서 만들어진 결과물 위에서 돌아가는 자바 프로그램
	      
 - 웹 서버: Nginx, Apache, IIS
	웹 서버는 클라이언트가 요청한 정적인 콘텐츠를 HTTP 프로토콜을 통하여 제공해주는 서버입니다.
	즉, 정적인 콘테츠란 정적페이지를 말한다.
	다른 역할로 동적인 요청이 클라이언트로부터 들어왔을 때,
	해당 요청을 웹 서버에서 처리할 수 없기 때문에 웹 컨테이너로 보내는 역할을 한다.	

 - 웹 컨테이너: Tomcat
	       동적인 데이터들을 처리하여 정적인 페이지로 생성해주는 소프트웨어 모듈이다.
	       
 - 웹 애플리케이션 서버(WAS: Web Application Server)
	: 웹 앱플리션과 서버 환경을 만들어 동작시키는 기능을 제공하는 소프트웨어 프레임워크이다.
	WAS는 동적 서버 콘텐츠를 수행하는 것이 일반적인 웹 서버와 구별되며,
	주로 데이터베이스 서버와 같이 수행한다. (웹 컨테이너 역할도 한다)
	WAS = WebServer + WebContainer(정적페이지+동적페이지)
	
 - 톰켓(WSA)
	: 포트가 8080포트로 오라클이랑 충돌이 난다.
	환경 설정할때 포트변경 및 한글처리를 해야 된다.
	Tomcat: 8080포트
	Oracle: 8080포트
	Apache: 80포트
	HTTP: 80포트
 - 톰켓 설치
	1. https://tomcat.apache.org/ Tomcat 9.0 버전 다운받기
	2. C:\에 zip 폴더 압축해제
	3. 환경설정
	   내 컴퓨터 우클릭 - 속성 - 고급 시스템 설정 - 환경변수 - 시스템 변수(새로만들기) - 
	   변수이름: CATALINA_HOME 변수값: 톰캣폴더위치 추가 -
	   Path 선택 후 편집 - %CATALINA_HOME%bin% 추가

	  포트변경 및 한글 처리하기
	  server.xml - 8080 찾기 - <Connector port="8090" URIEncoding="UTF-8" protocol="HTTP/1.1" 로 변경
	  logging.properties - java.util.logging.ConsoleHandler.encoding = UTF-8 => EUC-KR로 변경
	4. 세부내용
	   bin: 명령어들이 모여 있는 폴더
	         startup.bat: Tomcat 실행
	   conf: 환경설정 폴더
	           server.xml => 포트변경 및 한글처리
	   webapps: 프로젝트 파일(WAR)을 넣어서 Tomcat 실행하면 웹 사이트가 되는 폴더
	5. 실행하기
		1. startup.bat => 웹브라우저 창에서 localhost:8090 실행 => 아파치 톰캣 화면이 나오면 성공
		2. webapps 안에 이름폴더 만들고 텍스트 파일과 그림을 넣어보자
		3. 브라우저에서 http://localhost:8090/폴더명/파일명.확장명
